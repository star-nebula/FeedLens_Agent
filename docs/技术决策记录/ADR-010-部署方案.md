# ADR-010: 部署方案 — MVP 纯本地 Python 运行

## 上下文

Prompt 明确说「本地 Python 直接运行」做 MVP，也要求输出部署方案。之前的决策 T10 定为「本地开发 + Docker Compose for v1.0 交付」，但用户在对齐时明确选择「MVP 纯本地不提 Docker」。

可选方案：
- **A: MVP 纯本地 Python 运行** — 不打包 Docker，不写 Docker 配置
- **B: MVP 本地 + Docker Compose 交付** — 开发时本地，交付时容器化
- **C: MVP 纯本地 + docker-compose.yml 备用** — 不作为交付物，只写参考配置

## 决策

**采用方案 A**：MVP 纯本地 Python 运行，不提 Docker。

## 理由

1. **用户明确决定**：在对齐过程中用户选择了「纯本地不打包」，这是用户的决策
2. **Prompt 约束**：「本地 Python 直接运行」
3. **MVP 速度优先**：频繁改代码、重启服务、看日志，本地运行迭代速度最快
4. **减少交付范围**：不提 Docker 减少一个交付物，聚焦核心功能

## 影响

- MVP 不需要 docker-compose.yml
- 所有组件（主 Agent + MCP Server + ChromaDB + SQLite + Streamlit）本地 Python 运行
- APScheduler cron job 在 Python 进程内触发
- MCP Server (search_web SSE) 在本地 :8100 端口监听
- MCP Server (push_notification stdio) 作为子进程运行

## 后续演进

Docker Compose 作为 P2（后续迭代方向），v1.0 交付时再考虑容器化。
