# ADR-009: 用户反馈机制 — 三级反馈 + EMA 偏好更新

| 元数据 | |
|--------|--------|
| **状态** | 已采纳 |
| **日期** | 2026-06-17 |
| **决策者** | 项目决策组 |
| **来源** | 决策 P7、独特点 8/11/17/63 |

## 上下文

反馈机制是 FeedLens 个性化筛选的核心驱动力。用户对简报条目的反馈直接影响后续排序偏好。

需要决策的维度：
1. **反馈粒度**：二元（like/dislike）vs 三级（+irrelevant）vs 四级（+valuable）
2. **偏好更新算法**：简单累加 vs EMA 平滑
3. **偏好向量架构**：单一向量 vs 正负分离（v_like + v_dislike）
4. **反馈处理模式**：同步阻塞 vs 异步子图

## 决策

| 维度 | 决策 | 来源 |
|------|------|------|
| 反馈粒度 | **三级：like / dislike / irrelevant** | 决策 P7 |
| 偏好更新算法 | **EMA 平滑更新** | 独特点 17 |
| 偏好向量架构 | **正负分离（v_like + v_dislike）** | 独特点 11 |
| 反馈处理模式 | **异步子 Agent（feedback_workflow）** | 独特点 8 |

**反馈权重差异化**：

| 反馈类型 | preference 调整 | 语义 |
|---------|----------------|------|
| like | +0.15 | 内容有价值，强化此类偏好 |
| dislike | -0.10 | 内容质量差，弱化此类偏好 |
| irrelevant | -0.15 | 不属于关注领域，从候选集移除 |

## 理由

**三级反馈**：
1. 项目描述明确区分「不相关」和「质量差」——irrelevant 表示「不属于关注领域」，与 dislike（属于但质量差）语义不同
2. irrelevant 信号让系统从候选集移除整类内容，偏好收敛速度远快于二元反馈
3. valuable 与 like 语义重叠，舍去

**EMA 平滑**：
1. 简单累加（like +1, dislike -1）会导致偏好剧烈波动（连续 3 次 dislike 可能使偏好突变）
2. EMA 用指数移动平均平滑更新，偏好变化渐进式，更符合人类偏好变化规律

**正负分离**：
1. 单一偏好向量无法区分「喜欢什么」和「不喜欢什么」
2. v_like / v_dislike 分别维护，排序时可以独立计算 cosine(item, v_like) 和 cosine(item, v_dislike)
3. 偏好清理更精准：权重 < 0.1 的关键词自动清理

**异步子 Agent**：
1. 反馈处理不应阻塞主 Agent 的下一轮简报生成
2. 反馈子 Agent 独立运行，异步更新偏好向量

## 影响

- 反馈子 Agent 有独立的 StateGraph，与主流程异步运行
- ChromaDB user_preference collection 存储 v_like 和 v_dislike 两个向量
- 偏好权重低于 0.1 的关键词自动清理（防止噪声干扰）
- feedback_bias 作为即时补偿，偏好向量 EMA 更新后 bias 归零
