### 一、项目定位
FeedLens 是一个基于 LangGraph 编排的智能信息简报 Agent，它通过多源信息采集、向量去重、个性化多因子排序及反思审查，为用户自动生成结构化的每日信息简报，并具备基于用户反馈的持续偏好学习能力。

---

### 二、共识点分析
- **LangGraph StateGraph 状态机编排**：所有文档均采用 LangGraph 的 StateGraph 作为核心工作流引擎，通过节点与边串联“采集 → 去重 → 排序 → 简报生成 → 反思审查 → 反馈学习”的完整闭环。（主要来源章节：各文档第2章 Agent工作流设计）
- **四层记忆架构体系**：统一划分短期记忆（内存/State）、长期记忆、情节记忆和语义记忆，分别存储当前会话状态、用户偏好向量、历史执行日志和领域知识。（主要来源章节：各文档第4章 记忆系统设计）
- **FC与MCP工具分层调用策略**：一致认为简单、高频、本地计算的工具（如RSS采集、文本摘要、本地排序）应采用 Function Calling；而涉及独立部署、跨进程复用或密钥管理的重型服务（如搜索引擎）应采用 MCP。（主要来源章节：各文档第3章 工具清单）
- **国产化轻量级技术栈**：技术选型高度一致，采用 DeepSeek/通义千问作为 LLM，ChromaDB 作为向量库，SQLite 作为关系型数据库，Streamlit 构建前端，APScheduler 实现定时调度。（主要来源章节：各文档第8章 技术栈选择）
- **多因子加权排序算法**：均采用线性加权的打分公式，融合“语义相似度 + 时间时效衰减 + 用户偏好权重”三大核心因子对条目进行排序。（主要来源章节：各文档第5章 排序算法设计）
- **向量去重与“同事件不同角度”区分**：均基于余弦相似度阈值进行去重，并专门设计了区分“真正重复”与“同事件不同角度”的机制（保留主条目，折叠相关报道）。（主要来源章节：各文档第6章 去重策略设计）

### 四、独特点提取
- **【命名实体校验(NER)辅助去重】**：来自【GLM - 6.去重策略设计】。在向量相似度粗筛后，引入命名实体重叠率与标题编辑距离进行二次校验，比单纯依靠向量阈值更精确，展现了深度的 NLP 工程化思维。
- **【Agent自主推送重大事件机制】**：来自【GPT - 9.自主推送机制】。突破“每日定时日报”的静态逻辑，由 Planner 根据反思评估的事件重要性自主决定是否立即推送，真正体现了“主动式 Agent”的业务价值。
- **【情节记忆向量化与语义检索】**：来自【GLM - 4.记忆系统设计】。不仅用 SQLite 记录历史执行日志，还将执行摘要向量化存入 ChromaDB，支持通过自然语言检索历史经验（如“上次跑某话题用户喜欢什么”），极大增强了经验复用能力。
- **【排序权重的动态自适应调整】**：来自【Kimi - 5.3节 动态调权机制】与【TRAE - 5.4节 权重动态调整】。当用户连续对某类内容点赞或点踩时，系统自动微调该话题在打分公式中的权重系数，使排序模型具备在线学习与自适应能力，而非固定的静态权重。
- **【跨类别配额防霸屏策略】**：来自【GLM - 5.排序算法设计】。在 Top-N 简报条目选取时，按用户关注话题数均分配额，避免单一热门话题挤占简报空间，有效保证了信息多样性与阅读体验。
- **【短期记忆超窗压缩与Redis缓存】**：来自【Perplexity - 4.记忆系统设计】。引入 Redis 作为短期记忆缓存载体，并提出对超出滑动窗口的早期对话内容进行“总结压缩”而非直接丢弃，更好地保留了长会话的上下文连贯性。
