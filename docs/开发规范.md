# FeedLens 开发规范

## 一、技术栈

- Agent 编排：LangGraph StateGraph
- LLM：DeepSeek Chat
- Embedding：bge-small-zh-v1.5
- 向量库：ChromaDB
- 关系库：SQLite + WAL
- 定时任务：APScheduler
- 搜索补充：MCP Server (SSE :8100)
- 推送服务：MCP Server (stdio)
- 界面：Streamlit

## 二、核心架构

- 六层：感知层 → 大脑层 → 规划层 → 工具层 → 记忆层 → 展示层
- 主 Agent 工作流：understand\_intent → planner → invoke\_sub\_agent → observe\_results → planner(再思考) → coordinator\_reflect → push\_notification → update\_memory
- ReAct 循环上限：3 次
- 节点命名：coordinator\_reflect（主 Agent 审查）/ brief\_quality\_check（简报 Agent 工具）

## 三、核心算法

- 排序公式：final\_score = w₁·similarity + w₂·recency + w₃·(preference + feedback\_bias) + w₄·importance
- 冷启动权重：w₁=0.40, w₂=0.25, w₃=0.10, w₄=0.25（反馈<3）
- 有反馈权重：w₁=0.30, w₂=0.20, w₃=0.40, w₄=0.10（反馈≥3）
- 去重策略：≥0.88重复，≤0.70保留，0.70-0.88 LLM裁决（最多20对）
- 反馈权重：like=+0.15, dislike=-0.10, irrelevant=-0.15

## 四、数据模型（核心表）

- users: goal\_text, topics, keywords, preferred\_sources, goal\_embedding
- sources: RSS 源
- raw\_items: 原始采集
- deduped\_items: category, keywords, importance
- briefs: content\_json, content\_md, quality\_score
- feedback: feedback\_type (like/dislike/irrelevant)
- user\_preferences: keyword, weight, vector\_id

## 五、开发约束

- max\_react\_cycles: 3
- max\_llm\_adjudications: 20
- 简报质量阈值: score < 0.7 重试（最多2次）
- 冷启动阈值: 反馈 < 3 条
- 偏好清理阈值: 权重 < 0.1

## 六、推送规则

- 定时推送：APScheduler CronTrigger
- 破例推送：score > 0.85 且时效 < 2h

## 七、业务闭环

Goal → LLM提取偏好 → APScheduler触发 → 主Agent编排 → 采集→排序→简报→审查 → 推送 → 用户反馈 → 反馈Agent异步更新偏好 → 影响下一轮排序
