# Change — 变更记录与优化文档

> 本目录记录 FeedLens 项目从 MVP 交付后的所有变更：性能优化实施、架构演进规划、版本日志、Bug 修复。

---

## 目录导航

| 目录 | 定位 | 何时查阅 |
|------|------|---------|
| [`01_性能优化/`](./01_性能优化/) | 基于日志驱动的系统化性能优化（规划 + 实施 detail） | 了解某次优化的改动细节、效果数据、回退策略 |
| [`02_架构演进规划/`](./02_架构演进规划/) | MVP 后的架构增强规划（借鉴 OpenClaw 等） | 了解项目的架构演进思路和未来方向 |
| [`changelog.md`](./changelog.md) | 版本变更日志（条目索引） | 快速浏览各版本改了什么 |
| [`bugfix.md`](./bugfix.md) | Bug 修复记录 | 排查历史问题、了解已知缺陷 |

---

## 阅读顺序建议

```
changelog.md           ← 先看版本概览，了解整体演进
    ↓
01_性能优化/README.md   ← 再看优化路线图和效果汇总
    ↓
01_性能优化/0x_*.md    ← 深入具体优化的实施细节
    ↓
02_架构演进规划/        ← 了解架构层面的未来规划
    ↓
bugfix.md              ← 需要排查问题时查阅
```

---

## 文档约定

- 性能优化 detail 文档按 `00`（总体规划）→ `01~07`（各子项）编号排列
- 架构演进规划按 `01~04` 编号排列，反映演进时间线
- changelog 仅保留版本条目索引，实施细节统一在 `01_性能优化/` 的 detail 文档中
